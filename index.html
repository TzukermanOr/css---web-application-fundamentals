01

מה זה CSS

CSS = Cascading Style Sheets - שפה שמגדירה את המראה של דפי HTML. בלי CSS כל דף נראה כמו מסמך לבן משעמם.

Cascading = מדורג - כשכמה כללים חלים על אותו אלמנט, יש סדר עדיפויות שקובע מי "מנצח".

הפרדת אחריות

HTML = מבנה ותוכן (מה יש בדף)  |  CSS = עיצוב ומראה (איך נראה)  |  JavaScript = התנהגות (מה עושה)

התחביר הבסיסי

כל חוק CSS בנוי מ-3 חלקים: selector (מי), property (מה), value (איך).

body {
    background-color: lightblue;
}
h1 {
    color: navy;
    margin-left: 20px;
}
p {
    text-decoration: underline;
}
חלקתפקידדוגמאות
Selectorבוחר אלמנטים לעיצובh1   .btn   #header
Propertyהתכונה שמשתנהcolor   font-size   margin
Valueהערך החדשred   16px   #3b82f6
02

3 שיטות הטמעת CSS

1. Inline CSS - בתוך התגית

<h1 style="color: blue; font-size: 24px;">כותרת</h1>
למה לא מומלץ

אין שימוש חוזר, קשה לתחזק, Specificity גבוה מדי (1000). משתמשים רק לבדיקות מהירות.

2. Internal CSS - בתוך <style>

<head>
    <style>
        h1 { color: darkblue; text-align: center; }
        p  { color: #333; line-height: 1.6; }
    </style>
</head>

שימושי לדף בודד עם עיצוב ייחודי. Specificity תלוי בסלקטור.

3. External CSS - קובץ נפרד (מומלץ!)

<head>
    <link rel="stylesheet" href="style.css">
</head>
למה זו הדרך המומלצת

הפרדה מלאה HTML/CSS, שימוש חוזר על מספר דפים, הדפדפן עושה cache, קל לתחזק.

סדר עדיפויות בין השיטות

Specificity - סדר עדיפויות

כלל הזהב

!important (10000) > Inline (1000) > ID # (100) > Class . (10) > Tag (1)

אם ה-specificity שווה, הכלל האחרון שנכתב גובר. !important זה anti-pattern - אם אתה צריך אותו, כנראה שיש בעיה בארכיטקטורה.

ID, Class, Div & Span

ID vs Class

#ID - תעודת זהות

ייחודי - פעם אחת בדף. סלקטור #name. Specificity: 100.

.Class - שם משפחה

שימוש חוזר על מספר אלמנטים. סלקטור .name. Specificity: 10. אלמנט יכול לקבל כמה classes.

<h1 id="main-title" class="header highlight">כותרת</h1>
<p class="highlight">שימוש חוזר בclass</p>
חוקי שמות

שמות id/class לא יכולים להתחיל בספרה. מותר: אותיות, מקף (-), קו תחתון (_), ספרות (לא בהתחלה).

Div vs Span

<div> - Block<span> - Inline
תופס שורה שלמה, שובר שורהתופס רק את הרוחב הנדרש
לקיבוץ אלמנטים (sections, cards)לעיצוב חלק מטקסט בתוך פסקה
מקבל width/heightלא מקבל width/height

Semantic HTML5 - חלופות ל-div

<header>כותרת עליונה</header>

<footer>תחתית</footer>

Opacity vs RGBA

הבדל קריטי

opacity: 0.5 משנה שקיפות של כל האלמנט כולל ילדים וטקסט.

rgba(0,0,0,0.5) משנה שקיפות רק של הצבע עצמו. לרוב עדיף rgba.

/* Gradient */
.hero {
    background: linear-gradient(135deg, #667eea, #764ba2);
}
/* Opacity - משפיע על הכל */
.faded { opacity: 0.5; }
/* RGBA - רק הרקע שקוף */
.glass { background: rgba(0,0,0,0.5); color: white; }

Box Model - מודל הקופסה

כל אלמנט HTML הוא קופסה מלבנית עם 4 שכבות:

MARGIN
BORDER
PADDING
CONTENT
div {
    width: 300px;          /* תוכן */
    padding: 20px;         /* ריפוד פנימי */
    border: 4px solid red; /* גבול */
    margin: 30px auto;     /* שוליים + מרכוז */
    border-radius: 8px;    /* פינות מעוגלות */
}
/* SHORTHAND - כיוון השעון: top right bottom left */
padding: 10px 20px 30px 40px;
margin: 0 auto;  /* מרכוז אופקי! */
box-sizing: border-box - חובה!

בברירת מחדל, width לא כולל padding ו-border (אז 300px + 20px*2 + 4px*2 = 348px!). עם box-sizing: border-box, ה-width כולל הכל ונשאר 300px. Best practice:

*, *::before, *::after { box-sizing: border-box; }

Border Styles

border: 2px solid #333;     /* קו רציף */
border: 2px dashed #e11d48;  /* מקווקו */
border: 2px dotted #7c3aed;  /* נקודות */
border-radius: 50%;         /* עיגול (אם width=height) */
box-shadow: 0 4px 15px rgba(0,0,0,0.2);

Flexbox & Grid

Flexbox - פריסה 1D

.container {
    display: flex;
    flex-direction: row;          /* row | column */
    justify-content: center;     /* ציר ראשי */
    align-items: center;         /* ציר משני */
    gap: 20px;                   /* רווח בין פריטים */
    flex-wrap: wrap;             /* גלישה לשורה חדשה */
}
.item {
    flex: 1;          /* grow:1 shrink:1 basis:0 */
    flex: 0 0 200px;  /* לא גדל, לא מתכווץ, 200px */
}
/* מרכוז מושלם */
.perfect-center {
    display: flex;
    justify-content: center;
    align-items: center;
    min-height: 100vh;
}

זווית סייבר - iframe ונגישות

iframe - סכנת Clickjacking

תוקף שם אתר לגיטימי (כמו בנק) בתוך iframe שקוף מעל אתר תמים. המשתמש חושב שהוא לוחץ על "פרס" אבל בפועל מבצע פעולה באתר הבנק.

הגנות: X-Frame-Options: DENY בתגובת השרת, sandbox attribute על ה-iframe, ו-CSP headers.

alt="" בתמונות - נגישות וחוק

בישראל חובה חוקית להנגיש אתרים. ה-attribute alt ב-<img> מתאר את התמונה לקוראי מסך. מי שמטעה או לא שם alt חשוף לתביעות. זה לא אופציונלי.

iframe להטמעת YouTube

ביוטיוב: לחצן ימני על הסרטון ← "העתקת קוד הטמעה". מדביקים את ה-iframe ב-HTML וזה עובד. אפשר גם להטמיע אתר שלם: <iframe src="..." ></iframe>. בסיס למתקפות, אז תמיד בדקו שהמקור אמין.
