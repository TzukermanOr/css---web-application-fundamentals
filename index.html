01

מה זה CSS ולמה צריך את זה

CSS = Cascading Style Sheets (גיליונות סגנון מדורגים). זו השפה שאחראית על כל המראה של דפי אינטרנט: צבעים, גדלים, פונטים, מרווחים, מיקום אלמנטים בדף, אנימציות, עיצוב לטלפונים - כל זה CSS.

בלי CSS, כל אתר ייראה כמו מסמך Word לבן עם פונט ברירת מחדל. כל הכפתורים, הצבעים, הפריסה, ה-hover effects - כל מה שהופך אתר ליפה ושמיש - זה CSS.

תחשבו על בנייה: HTML זה השלד (קירות, רצפה, תקרה). CSS זה העיצוב הפנימי (צבע קירות, ריצוף, תאורה, רהיטים). JavaScript זה האלקטרוניקה (מנורות שנדלקות, דלתות אוטומטיות).

הפרדת אחריות - Separation of Concerns

HTML = מבנה ותוכן (מה יש בדף)  |  CSS = עיצוב ומראה (איך הדף נראה)  |  JavaScript = התנהגות (מה הדף עושה)

מה זה "Cascading"?

Cascading = מדורג. כשכמה כללי CSS חלים על אותו אלמנט, הדפדפן צריך להחליט מי "מנצח". יש לו מערכת ניקוד שנקראת Specificity (ספציפיות). הכלל הספציפי ביותר גובר. למשל, אם כתבתם p { color: blue; } וגם #special { color: red; }, ופסקה מסוימת יש לה את ה-id "special" - היא תהיה אדומה כי ID ספציפי יותר מ-tag.

התחביר הבסיסי - CSS Syntax

כל חוק CSS בנוי מ-3 חלקים: Selector (מי מקבל את העיצוב), Property (מה משתנה), Value (לאיזה ערך).

CSS 
/* selector = מי מקבל את העיצוב */
/* property = מה משתנה */
/* value = לאיזה ערך */

body {
    background-color: lightblue;   /* צבע רקע של כל הדף */
    font-family: Arial, sans-serif; /* פונט לכל הדף */
    margin: 0;                      /* הסרת מרווח ברירת מחדל */
}

h1 {
    color: navy;                  /* צבע הטקסט */
    font-size: 32px;              /* גודל הפונט */
    text-align: center;           /* יישור למרכז */
    margin-bottom: 20px;          /* מרווח למטה */
}

p {
    color: #333333;               /* צבע אפור כהה */
    line-height: 1.6;             /* גובה שורה - קריאות טובה */
    text-decoration: underline;   /* קו תחתון */
}

/* הערה: הערות ב-CSS נכתבות בין /* ו-*/ */
חלק תפקיד דוגמאות
Selector בוחר אלמנטים לעיצוב - "מי?" h1 .btn #header div p
Property התכונה שמשתנה - "מה?" color font-size margin background
Value הערך החדש - "איך?" red 16px center #3b82f6

שימו לב לתחביר: כל property וערך מופרדים ב-: (נקודתיים). כל הצהרה נגמרת ב-; (נקודה-פסיק). כל הכללים של סלקטור עטופים בסוגריים מסולסלים { }.

למה CSS חשוב - 4 יתרונות מרכזיים

1. Grouping - עיצוב קבוצתי: אפשר לעצב כמה אלמנטים בו-זמנית. במקום לעבור על כל אלמנט בנפרד, כותבים h1, h2, h3 { color: navy; } וכל הכותרות מקבלות את אותו צבע.

2. Avoid Duplication - מניעת כפילות: קובץ CSS אחד יכול לשרת 100 דפי HTML שונים. שינוי צבע ראשי בקובץ אחד = שינוי בכל האתר.

3. Clean Code - קוד נקי: כשהעיצוב מופרד מה-HTML, הקוד הרבה יותר קריא. במקום תגיות HTML מבולגנות עם style ארוך בכל אחת, יש קוד HTML נקי וקובץ CSS מסודר בצד.

4. Caching - שמירה בזיכרון: הדפדפן שומר את קובץ ה-CSS בזיכרון (cache). כשהמשתמש עובר לדף אחר באתר, הדפדפן לא צריך להוריד את ה-CSS שוב - הדף נטען מהר יותר.

HTML vs CSS - עיצוב טקסט

ב-HTML יש תגיות עיצוב בסיסיות, אבל CSS יכול לעשות את אותו דבר בצורה טובה יותר ועם יותר שליטה:

HTML (ישן) CSS (עדיף!) מה עושה
<b> font-weight: bold; טקסט מודגש
<i> font-style: italic; טקסט נטוי
<u> text-decoration: underline; קו תחתון
<mark> background-color: yellow; הדגשת רקע
<small> font-size: smaller; טקסט קטן
הערה חשובה על גרסאות CSS

אין דבר שנקרא "CSS4" כגרסה אחת רשמית. מאז CSS3, ה-W3C (הארגון שמפתח את התקנים) מפתח CSS כאוסף מודולים עצמאיים - כל אחד בגרסה משלו. למשל: CSS Selectors Level 4, CSS Grid Level 2, CSS Color Level 4. אז אם מישהו אומר "CSS4 שוחרר ב-2017" - זה לא מדויק.

02

3 שיטות להטמעת CSS בדף HTML

יש 3 דרכים "להזריק" CSS לתוך דף HTML. כל שיטה עם יתרונות וחסרונות. חשוב להכיר את כולן כי תפגשו את שלושתן בקוד אמיתי.

שיטה 1: Inline CSS - עיצוב בתוך התגית

כותבים את ה-CSS ישירות על אלמנט ספציפי, בתוך attribute שנקרא style. כל אלמנט מעוצב בנפרד.

HTML 
<!-- כל אלמנט מקבל עיצוב בנפרד -->
<h1 style="color: blue; font-size: 24px; text-align: center;">
    כותרת כחולה
</h1>

<p style="background-color: yellow; padding: 10px; border-radius: 8px;">
    פסקה עם רקע צהוב
</p>

<h1 style="background-color: peru; color: white; padding: 15px;">
    I have a dream!
</h1>
למה Inline CSS לא מומלץ

Specificity גבוה מדי (1000): קשה מאוד לדרוס inline CSS עם CSS אחר. זה יוצר בעיות כשרוצים לשנות עיצוב אחר כך.

אין שימוש חוזר: אם 10 פסקאות צריכות אותו צבע, צריך לכתוב את אותו style 10 פעמים.

לא ניתן להגדיר hover או animation: ב-inline CSS אי אפשר לכתוב :hover או @keyframes.

קוד מבולגן: ה-HTML הופך לקשה לקריאה כשכל תגית מלאה בעיצוב.
מתי כן להשתמש: רק לבדיקות מהירות, או כשצריך לדרוס סגנון מאוד ספציפי בנסיבות חריגות.

שיטה 2: Internal CSS - בתוך תגית <style>

כותבים את כל ה-CSS בתוך תגית <style> שנמצאת ב-<head> של המסמך. העיצוב חל על הדף הנוכחי בלבד - לא משפיע על דפים אחרים.

HTML 
<!DOCTYPE html>
<html lang="he" dir="rtl">
<head>
    <meta charset="UTF-8">
    <title>הדף שלי</title>

    <!-- כל ה-CSS במקום אחד, בתוך style -->
    <style>
        /* עיצוב כללי לדף */
        body {
            background-color: #f0f4f8;
            font-family: Arial, sans-serif;
            color: #1e293b;
        }

        /* עיצוב כותרות */
        h1 {
            color: darkblue;
            text-align: center;
            font-size: 2em;
        }

        /* עיצוב פסקאות */
        p {
            color: #333;
            line-height: 1.6;
            font-family: cursive;
        }

        /* עיצוב לפי ID */
        #login-header {
            color: green;
            font-size: 30px;
        }

        /* עיצוב לפי Class */
        .third-header {
            color: blueviolet;
            font-size: 24px;
        }
    </style>
</head>
<body>
    <h1>good morning</h1>
    <h2 id="login-header">login page</h2>
    <h3 class="third-header">welcome</h3>
    <p>טקסט רגיל</p>
</body>
</html>

מתי להשתמש: כשיש דף בודד עם עיצוב ייחודי שלא חוזר על עצמו.
כלל אצבע: אם אותו סגנון מופיע ב-3 דפים או יותר - תעבירו אותו לקובץ חיצוני (שיטה 3).

שיטה 3: External CSS - קובץ נפרד (הדרך המומלצת!)

יוצרים קובץ נפרד עם סיומת .css ומקשרים אותו ל-HTML עם תגית <link>. זו הדרך הסטנדרטית והמומלצת בתעשייה.

קובץ HTML (index.html):

index.html 
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>האתר שלי</title>

    <!-- קישור לקובץ CSS חיצוני -->
    <link rel="stylesheet" href="style.css">

    <!-- אפשר לקשר מספר קבצי CSS -->
    <link rel="stylesheet" href="layout.css">
    <link rel="stylesheet" href="colors.css">
</head>
<body>
    <h1 id="alpha" class="best">better design</h1>
    <p id="par">i think it is a better design</p>
</body>
</html>

קובץ CSS (style.css):

style.css 
/* עיצוב כללי */
body {
    background-color: aquamarine;
    font-family: "Segoe UI", sans-serif;
    margin: 0;
    padding: 20px;
}

/* סלקטור ID - עם # */
#alpha {
    background-color: blue;
    color: white;
    padding: 15px;
}

/* סלקטור Class - עם נקודה */
.best {
    font-family: fantasy;
    font-size: 28px;
}
למה External CSS הדרך המומלצת

הפרדה מלאה: HTML נשאר נקי עם תוכן בלבד, CSS בקובץ נפרד.

שימוש חוזר: אותו קובץ style.css משרת את כל דפי האתר. שינוי אחד = שינוי בכל האתר.

Cache: הדפדפן שומר את ה-CSS בזיכרון. בפעם הבאה שהמשתמש נכנס לדף אחר באתר, ה-CSS כבר טעון - מהיר יותר.

עבודת צוות: מעצב יכול לעבוד על ה-CSS בזמן שמתכנת עובד על ה-HTML - בלי לדרוך אחד על השני.

אם אותו אלמנט מקבל עיצוב מכל 3 השיטות בו-זמנית, מי מנצח?

ובתוך אותו מקור - הכלל האחרון שנכתב גובר. הדפדפן קורא את ה-CSS מלמעלה למטה, ואם יש 2 כללים עם אותו specificity, האחרון מנצח.

03

Selectors - סלקטורים (כל הסוגים)

סלקטורים הם "השאילתות" שלנו - הדרך לבחור אלמנטים ב-HTML ולהחיל עליהם עיצוב. זה הלב של CSS. ככל שתבינו סלקטורים טוב יותר, כך תוכלו לכתוב CSS מדויק ויעיל יותר.

הכניסו סלקטור CSS ותראו את ציון ה-Specificity שלו. נסו: #nav .item a:hover או div.container > p.intro

כלל הזהב של Specificity

!important (10000) > Inline style (1,0,0,0) > ID # (0,1,0,0) > Class/Pseudo-class/Attribute (0,0,1,0) > Element/Pseudo-element (0,0,0,1).
אם שווה - הכלל האחרון גובר. !important הוא anti-pattern - אם אתם צריכים אותו, כנראה שמשהו שבור בארכיטקטורה.

04

ID, Class, Div & Span - מבנה בסיסי

ID = תעודת זהות. ייחודי - פעם אחת בדף. סלקטור: #name. Specificity: 100. משמש גם כ-anchor לניווט (href="#section1") וב-JavaScript (document.getElementById()).

Class = שם משפחה. שימוש חוזר על מספר אלמנטים. סלקטור: .name. Specificity: 10. אלמנט אחד יכול לקבל כמה classes: class="btn big red".

HTML 
<body>
    <!-- id = ייחודי. class = שימוש חוזר -->
    <h1 id="first" class="head">First Header</h1>
    <h1 id="second" class="head">Second Header</h1>
    <!-- שני ה-h1 חולקים class="head" אבל כל אחד עם id ייחודי -->

    <!-- אלמנט עם כמה classes -->
    <p class="text-large text-bold text-blue">שלושה classes!</p>
</body>
חוקי שמות

שמות id/class לא יכולים להתחיל בספרה. מותרים: אותיות, מקף (-), קו תחתון (_), ספרות (לא בהתחלה). שיטת BEM מומלצת: .card__title--large.

Div vs Span - ההבדל המרכזי

<div> = Block. תופס שורה שלמה, שובר שורה, מקבל width/height. משמש לקיבוץ אלמנטים (sections, כרטיסים, layouts).

<span> = Inline. תופס רק רוחב נדרש, לא שובר שורה, לא מקבל width/height. משמש לעיצוב חלק מטקסט בתוך פסקה.

HTML 
<!-- div = block. תופס שורה שלמה -->
<div style="background-color: gray;">
    <h1>Im Inside Division</h1>
    <p>Im <span style="color: white;">Inside</span> This Division too</p>
</div>
<!-- span = inline. עיצוב חלק מטקסט בלי לשבור שורה -->

<!-- Semantic HTML5 - חלופות ל-div -->
<header>כותרת עליונה</header>
<nav>ניווט</nav>
<main>תוכן ראשי</main>
<section>חלק</section>
<article>מאמר</article>
<aside>תוכן צדדי</aside>
<footer>תחתית</footer>

Block elements: div, h1-h6, p, section, article, header, footer, form, ul, ol, li, nav, main
Inline elements: span, a, img, strong, em, input, button, label, br

opacity vs rgba - הבדל קריטי!

opacity: 0.5 - משנה שקיפות של כל האלמנט, כולל הטקסט והילדים שלו. הטקסט גם נראה שקוף.

rgba(0,0,0,0.5) - משנה שקיפות רק של הצבע עצמו. הטקסט נשאר אטום. לרוב rgba עדיף.

CSS 
/* Linear Gradient - מעבר צבעים ליניארי */
.hero {
    background: linear-gradient(135deg, #667eea, #764ba2);
}

/* Gradient עם כמה צבעים */
.rainbow {
    background: linear-gradient(to right, red, orange, yellow, green, blue);
}

/* Radial Gradient - מעבר עיגולי */
.circle {
    background: radial-gradient(circle, #ff0000, #0000ff);
}

/* RGBA - רק הרקע שקוף, הטקסט אטום */
.glass {
    background-color: rgba(0, 0, 0, 0.5);
    color: white;  /* הטקסט לא מושפע! */
}

/* Opacity - כל האלמנט שקוף (כולל טקסט) */
.faded {
    opacity: 0.5;  /* 0=שקוף, 0.5=חצי, 1=אטום */
}
/* שימו לב: אלמנט עם opacity:0 עדיין תופס מקום בדף! */

06

טיפוגרפיה ויחידות מידה

CSS 
h1 {
    font-family: Georgia, "Times New Roman", serif; /* גופן + fallbacks */

padding: 10px 20px 30px 40px;        /* top, right, bottom, left */

box-sizing: border-box - חובה להבין!

בברירת מחדל (content-box): width: 300px + padding: 20px (שני צדדים) + border: 5px (שני צדדים) = 350px בפועל!

עם box-sizing: border-box - ה-width כולל הכל ונשאר 300px. Best practice:

*, *::before, *::after { box-sizing: border-box; }

Margin Collapsing: כששני block elements נפגשים (margin-bottom של אחד + margin-top של השני), הם לא מצטברים - רק הגדול נלקח. 30px + 20px = 30px, לא 50px!

Flexbox - פריסה גמישה חד-ממדית

Flexbox מסדר אלמנטים בשורה אחת (או עמודה אחת). מושלם לתפריטים, שורות כפתורים, מרכוז, וכרטיסים בשורה.

CSS Grid - פריסת רשת דו-ממדית

Grid מאפשר שליטה בשני ממדים בו-זמנית (שורות + עמודות). מושלם לדפים שלמים, גלריות, דשבורדים.

CSS 
.grid {
    display: grid;
    grid-template-columns: repeat(3, 1fr);   /* 3 עמודות שוות */
    grid-template-columns: 200px 1fr 200px;  /* sidebar - main - sidebar */
    grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
    /* ^ שורת קסם! responsive אוטומטי - עמודות שמתאימות לרוחב */
    gap: 20px;
}

.wide { grid-column: span 2; }  /* תופס 2 עמודות */
.tall { grid-row: span 3; }     /* תופס 3 שורות */

/* Grid Areas - שיטה ויזואלית */
.layout {
    display: grid;
    grid-template-areas:
        "header header header"
        "sidebar main   main"
        "footer footer footer";
    grid-template-columns: 250px 1fr 1fr;
    grid-template-rows: auto 1fr auto;
    min-height: 100vh;
}
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }
Flexbox vs Grid - מתי משתמשים במה?

Flexbox = כיוון אחד (שורה או עמודה): שורת כפתורים, תפריט ניווט, מרכוז אלמנטים. Grid = שני ממדים (שורות + עמודות): דף שלם, גלריית תמונות, dashboard. אפשר ורצוי לשלב! Grid ל-layout הכללי, Flexbox לרכיבים בתוכו.

זווית סייבר - Clickjacking

תוקף שם אתר לגיטימי (כמו בנק) בתוך iframe שקוף לחלוטין (opacity: 0) מעל אתר שנראה תמים ("לחץ כאן לפרס!"). המשתמש חושב שלוחץ על משחק, אבל בפועל לוחץ על "העבר כסף" באתר הבנק.

הגנות: X-Frame-Options: DENY (השרת חוסם הטמעה), sandbox attribute על ה-iframe, CSP headers. תמיד בדקו שהמקור אמין.

נגישות - alt בתמונות (חובה חוקית!)

בישראל יש חוק נגישות. ה-attribute alt ב-<img> מתאר את התמונה לאנשים שמשתמשים בקוראי מסך. מי שלא שם alt או שמטעה - חשוף לתביעות. זה לא אופציונלי. דוגמה: <img src="laptop.jpg" alt="מחשב נייד שחור 15 אינץ'">
