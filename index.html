01

מה זה CSS ולמה צריך את זה

CSS = Cascading Style Sheets (גיליונות סגנון מדורגים). זו השפה שאחראית על המראה של דפי אינטרנט. צבעים, גדלים, פונטים, מרווחים, מיקום אלמנטים בדף, אנימציות - כל זה CSS. בלי CSS כל אתר ייראה כמו מסמך Word לבן עם פונט ברירת מחדל.

תחשבו על בנייה: HTML זה השלד (הקירות, הרצפה, התקרה). CSS זה העיצוב הפנימי (צבע הקירות, סוג הריצוף, תאורה). JavaScript זה האלקטרוניקה (מנורות שנדלקות, דלתות אוטומטיות).

הפרדת אחריות - Separation of Concerns

HTML = מבנה ותוכן (מה יש בדף)  |  CSS = עיצוב ומראה (איך נראה)  |  JavaScript = התנהגות (מה עושה)

Cascading = מדורג - כשכמה כללי CSS חלים על אותו אלמנט, הדפדפן מחשב סדר עדיפויות (Specificity) ובוחר את המנצח. כלומר, CSS שנמצא "קרוב יותר" לאלמנט או שהוא ספציפי יותר - ידרוס CSS אחר.

למה CSS חשוב? אפשר לשנות את כל המראה של אתר שלם ע"י עריכת קובץ CSS אחד. אפשר לתת לאותו HTML מראה שונה לגמרי במובייל לעומת דסקטופ. והדפדפן שומר (cache) את קובץ ה-CSS, אז טעינה חוזרת מהירה יותר.

התחביר הבסיסי

כל חוק CSS בנוי מ-3 חלקים:

CSS 
/* selector = מי מקבל עיצוב */
/* property = מה משתנה */
/* value = לאיזה ערך */

body {
    background-color: lightblue;   /* צבע רקע */
}

h1 {
    color: navy;                  /* צבע טקסט */
    font-size: 32px;              /* גודל פונט */
    text-align: center;           /* יישור למרכז */
    margin-left: 20px;            /* מרווח שמאלי */
}

p {
    text-decoration: underline;   /* קו תחתון */
    line-height: 1.6;             /* גובה שורה */
}
חלקתפקידדוגמאות
Selectorבוחר אלמנטים לעיצוב - "מי?"h1 .btn #header div p
Propertyהתכונה שמשתנה - "מה?"color font-size margin background
Valueהערך החדש - "איך?"red 16px center #3b82f6

יתרונות CSS

Grouping - אפשר לעצב כמה אלמנטים בו-זמנית: h1, h2, h3 { color: navy; }.
Avoid Duplication - קובץ CSS אחד משרת כמה דפי HTML. שינוי אחד = שינוי בכל האתר.
Clean Code - כשהעיצוב מופרד מה-HTML, הקוד קל יותר לקריאה ולתחזוקה.
Caching - הדפדפן שומר את קובץ ה-CSS בזיכרון ולא מוריד אותו שוב בכל דף חדש.

שימו לב

אין דבר שנקרא "CSS4" כגרסה אחת. מאז CSS3, ה-W3C מפתח CSS כאוסף מודולים עצמאיים (CSS Selectors Level 4, CSS Grid Level 2, וכו'). כל מודול בגרסה משלו. אז אל תתייחסו ל-"CSS4" כדבר רשמי.

02

3 שיטות הטמעת CSS

יש 3 דרכים "להזריק" CSS לתוך דף HTML. כל שיטה עם יתרונות וחסרונות. נעבור על כולן:

שיטה 1: Inline CSS - בתוך התגית

כותבים את ה-CSS ישירות על האלמנט, בתוך attribute שנקרא style. כל אלמנט מעוצב בנפרד.

HTML 
<h1 style="color: blue; font-size: 24px;">כותרת כחולה</h1>
<p style="background-color: yellow; padding: 10px;">פסקה צהובה</p>
למה לא מומלץ

Specificity של 1000 (קשה לדרוס), אין שימוש חוזר (צריך לחזור על הקוד בכל אלמנט), לא ניתן להגדיר hover או animation, קוד מבולגן. שימושי רק לבדיקות מהירות.

שיטה 2: Internal CSS - בתוך <style>

כותבים CSS בתוך תגית <style> ב-<head>. חל על הדף הנוכחי בלבד. הרבה יותר מסודר מ-inline.

HTML 
<head>
  <style>
    h1 {
      color: darkblue;
      text-align: center;
    }
    p {
      color: #333;
      line-height: 1.6;
      font-family: Arial, sans-serif;
    }
  </style>
</head>

כלל אצבע: אם אותו סגנון מופיע ב-3 דפים ומעלה - תעבירו אותו לקובץ חיצוני.

שיטה 3: External CSS - קובץ נפרד (מומלץ!)

יוצרים קובץ נפרד עם סיומת .css ומקשרים אותו ל-HTML עם תגית <link>.

HTML 
<head>
  <!-- אפשר לקשר מספר קבצי CSS -->
  <link rel="stylesheet" href="style.css">
  <link rel="stylesheet" href="layout.css">
</head>
למה זו הדרך המומלצת

הפרדה מלאה HTML/CSS, שימוש חוזר על מספר דפים, הדפדפן עושה cache (טעינה מהירה), קל לתחזק, עובד עם צוותים.

סדר עדיפויות בין 3 השיטות

1. Inline
מנצח תמיד (1000)
2. Internal
מנצח External
3. External
עדיפות בסיסית

אם ה-specificity זהה, הכלל האחרון שנכתב גובר (הדפדפן קורא מלמעלה למטה).

04

ID, Class, Div & Span

#ID - תעודת זהות

ייחודי - פעם אחת בדף. סלקטור: #name. Specificity: 100. משמש גם כ-anchor (href="#section") וב-JS (getElementById).

.Class - שם משפחה

שימוש חוזר על מספר אלמנטים. סלקטור: .name. Specificity: 10. אלמנט יכול לקבל כמה classes: class="btn big red".

HTML 
<!-- ID = ייחודי, Class = שימוש חוזר -->
<h1 id="main-title" class="header highlight">כותרת</h1>
<p class="highlight">שימוש חוזר ב-class</p>
<p class="highlight bold">שני classes על אותו אלמנט</p>
חוקי שמות

שמות id/class לא יכולים להתחיל בספרה. מותרים: אותיות, מקף (-), קו תחתון (_), ספרות (לא בהתחלה). שיטת BEM מומלצת: .card__title--large.

Div vs Span

<div> = Block - תופס שורה שלמה, שובר שורה, מקבל width/height. לקיבוץ אלמנטים (sections, כרטיסים).
<span> = Inline - תופס רק רוחב נדרש, לא שובר שורה, לא מקבל width/height. לעיצוב חלק מטקסט בתוך פסקה.

Block: div, h1-h6, p, section, article, header, footer, form, ul, ol, li, nav, main
Inline: span, a, img, strong, em, input, button, label, br

Semantic HTML5

במקום div לכל דבר, יש אלמנטים סמנטיים: <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>. הם מתנהגים כמו div אבל נותנים משמעות לדפדפן, לקוראי מסך ול-SEO.

07

Box Model - מודל הקופסה

כל אלמנט HTML הוא קופסה מלבנית עם 4 שכבות. הבנת Box Model = הבסיס לכל layout.

MARGIN
BORDER
PADDING
CONTENT

Content = הטקסט/תמונה. Padding = ריפוד פנימי (הרקע מכסה אותו). Border = הגבול. Margin = שוליים חיצוניים (הרקע לא מכסה).

CSS 
div {
    width: 300px;   height: 200px;
    padding: 20px;          /* ריפוד פנימי */
    border: 4px solid red;  /* גבול */
    margin: 30px auto;      /* שוליים + מרכוז */
    border-radius: 12px;   /* פינות מעוגלות */
}
/* Shorthand - כיוון השעון: top right bottom left */
padding: 10px 20px 30px 40px;
margin: 0 auto;  /* מרכוז אופקי! */
/* Border styles: solid|dashed|dotted|double|none */
border-radius: 50%;  /* עיגול (אם width=height) */
box-shadow: 0 4px 15px rgba(0,0,0,0.2);
box-sizing: border-box - חובה!

בברירת מחדל: width לא כולל padding ו-border. אז width:300px + padding:20px + border:5px = 350px!

עם box-sizing: border-box ה-width כולל הכל. Best practice: *, *::before, *::after { box-sizing: border-box; }

Position, Display & Overflow

staticברירת מחדל. בזרימה. top/left לא עובדיםכל אלמנט רגיל

Display: block (שורה שלמה) | inline (רוחב מינימלי, לא מקבל w/h) | inline-block (inline שמקבל w/h) | none (נעלם) | flex | grid

visibility: hidden = לא נראה אבל תופס מקום. display: none = נעלם לגמרי.

z-index - שכבתיות. עובד רק עם position שאינו static! ערך גבוה = מלפנים.

CSS Grid - פריסה 2D

CSS 
.grid {
    display: grid;
    grid-template-columns: repeat(3, 1fr);  /* 3 עמודות שוות */

Flexbox = כיוון אחד (שורה/עמודה). Grid = שני ממדים (שורות + עמודות). אפשר לשלב!

זווית סייבר - iframe & Clickjacking

תוקף שם אתר לגיטימי בתוך iframe שקוף מעל אתר תמים. המשתמש חושב שלוחץ על "פרס" אבל מבצע פעולה באתר הבנק. הגנות: X-Frame-Options: DENY, sandbox attribute, CSP headers. תמיד בדקו שהמקור אמין לפני הטמעת iframe.
